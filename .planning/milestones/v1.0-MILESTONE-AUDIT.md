### Key Architectural Patterns Verified

1. **Event bus nervous system:** All components communicate via shared EventBus singleton
2. **Dual-LLM privilege separation:** SanitizedOutput is the only data that crosses the trust boundary
3. **4 isolated Gemini clients:** Defense, Commentary, Scoring, Deliberation each have separate clients
4. **Shared DisplayServer:** Single instance serves commentary, scoring, and deliberation display
5. **Python-authoritative scores:** Weighted totals computed in Python, never trusting LLM arithmetic
6. **Detached task pattern:** Score reveals and display pushes never block the main event bus
