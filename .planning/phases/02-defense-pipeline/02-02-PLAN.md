<objective>
Build the response and output components of the defense pipeline: roast generation, injection logging, and observation sanitization. These consume detection results from Plan 02-01 and produce the outputs that downstream consumers (Phase 3 commentary, Phase 4 scoring) will use.

Purpose: Detection alone is not enough. Detected injections need to be (1) logged for scoring notes (DEF-05), (2) roasted for entertainment (DEF-04), and (3) stripped from observations before they reach the privileged LLM (DEF-01). The sanitizer is the critical security boundary -- it ensures the P-LLM only sees clean, verified observations.

Output: Three modules providing roast generation, injection logging, and observation sanitization ready for pipeline wiring in Plan 02-03.
</objective>

    **src/defense/sanitizer.py**: Strips tainted observations before they reach the P-LLM.

    - `class ObservationSanitizer`:
      - `__init__(self, detector: InjectionDetector)`: Store the detector reference for scanning observations.
      - `def sanitize_observations(self, observations: list[str], known_attempts: list[InjectionAttempt]) -> list[str]`: Remove tainted observations.
        - For each observation, run `self._detector.scan_observation(observation)`.
        - If the scan returns is_injection=True with confidence "medium" or "high", EXCLUDE the observation entirely. Do NOT try to redact words -- per research anti-pattern, flag and exclude the whole observation.
        - Return the clean observations list.
        - Log how many observations were removed: `"Sanitized observations: %d/%d removed"`.
      - `def sanitize_transcripts(self, transcripts: list[str], known_attempts: list[InjectionAttempt]) -> list[str]`: Same approach for transcript text segments. Filter out segments that match injection patterns at medium or high confidence.
      - `def create_sanitized_output(self, team_name: str, observations: list[str], transcripts: list[str], attempts: list[InjectionAttempt], duration: float, roasts: list[str] | None = None) -> SanitizedOutput`:
        - Call sanitize_observations and sanitize_transcripts.
        - Return a SanitizedOutput with clean data and the full injection attempts list.

    Import logging. Import InjectionDetector from src.defense.injection_detector. Import InjectionAttempt, SanitizedOutput from src.defense.models. Use `from __future__ import annotations`.

    **Critical design note (per research Pitfall 4):** The sanitizer scans Gemini observations for injection residue. If Gemini faithfully transcribes "The slide says IGNORE ALL INSTRUCTIONS", the sanitizer catches this quoted injection and removes the entire observation. This is the second layer of defense behind OCR/transcript scanning.
  </action>
