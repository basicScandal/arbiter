**CSS additions (add to existing style block):**

Score container styling:
- `#score-card`: Fixed position overlay or section below commentary. Dark card background (#1a1a2e or similar dark theme matching existing), rounded corners, padding, max-width 600px centered, hidden by default (display: none)
- `.score-header`: Team name in large bold text, uppercase, with a glow effect (text-shadow with cyan/green hacker aesthetic)
- `.score-criterion`: Individual criterion row with name, score bar, and justification. Use flexbox for layout. Score value displayed prominently. Include a CSS transition for opacity and transform (slide in from left, 0.5s ease-out)
- `.score-bar`: Visual bar showing score out of 10. Width transitions from 0 to (score/10 * 100)%. Background gradient from red (low) through yellow to green (high). CSS transition on width: 1s ease-out
- `.score-total`: Final total score display. Large font (3-4rem), centered, with a pulse animation (scale 1 -> 1.1 -> 1, repeated). Color based on score range: red (<4), yellow (4-6), green (6-8), gold (>8)
- `.score-justification`: Small italic text below each criterion, max 2 lines with ellipsis overflow
- Animation keyframes: `@keyframes slideIn` (translateX(-30px) opacity 0 -> translateX(0) opacity 1), `@keyframes pulse` (scale(1) -> scale(1.15) -> scale(1)), `@keyframes glowPulse` for header text shadow

**JavaScript additions (add to existing WebSocket onmessage handler):**

Handle new message types in the existing switch/if-else on `data.type`:

- `score_intro`: Show the score-card container. Set the score-header text to team name. Clear any previous criteria. Add a "SCORING" label or similar. Use CSS class to trigger fade-in animation.

- `score_criterion`: Append a new criterion row to the score-card. Each row shows: criterion name (left), score as number (right), a visual bar filling to the score width, and justification text below. - On "clear" message (already handled): Also hide the score-card and clear its contents.

**Important design notes:**
- The score card should NOT overlap with commentary text -- it should appear in its own section (below commentary or as a distinct visual area)
- Use the dark-theme aesthetic already established in the display (examine the existing HTML for color scheme)
- Scores should be readable from the back of a venue -- large fonts, high contrast
- The score bar animation should be visually satisfying: starts at 0 width and expands to the score value
