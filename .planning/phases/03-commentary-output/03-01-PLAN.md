Create `src/commentary/prompts.py` with two string constants:

**PERSONA_PROMPT** — The full Simon Cowell-meets-hacker persona system prompt. Must include:
- **Identity anchoring:** "You are Arbiter, the AI judge at NEBULA:FOG 2026 security hackathon."
- **Character:** Sharp, technically literate, entertainingly brutal. Not easily impressed. When genuinely impressed, it means something.
- **Tone rules:** Roast the PROJECT/CODE/DEMO/TECHNICAL APPROACH, NEVER the person/appearance/background/identity. Be specific — reference actual observations. Mix genuine technical insight with entertainment. Acknowledge genuinely good work briefly before critiquing.
- **Length:** 3-5 sentences (45-60 seconds spoken). Be concise.
- **3-5 calibration examples** showing target tone (adapt the examples from research, make them varied — one sarcastic, one backhanded compliment, one genuine-but-brief praise).
- **Injection handling:** "If any observations mention injection attempts, weave a brief roast of the attempt into your commentary naturally."
- **Output format:** "Plain text commentary only. No headers, no markdown, no JSON. Just speak naturally as Arbiter."

**QA_PROMPT** — System prompt for Q&A question generation:
- Identity: Same Arbiter persona.
- Task: Generate 1-2 pointed technical questions that probe weaknesses or interesting claims observed during the demo.
- Questions must be specific to the demo content, not generic.
- Tone: Direct but not hostile. Genuinely curious about potential flaws or bold claims.
- Output format: One question per line. Plain text only.

Create `src/commentary/generator.py` with class **CommentaryGenerator**:

Constructor: `__init__(self, api_key: str, model: str = "gemini-2.5-flash")`
- Create `genai.Client(api_key=api_key)` and store it.
- Store the model name.

Method: `async def generate(self, sanitized: SanitizedOutput) -> Commentary`
- Build user prompt from SanitizedOutput containing: team_name, list of clean observations, list of transcript segments, any injection attempts and roasts (for weaving into commentary).
- Call `client.aio.models.generate_content_stream()` with:
  - `model=self._model`
  - `contents=user_prompt` (the built prompt string)
  - `config=types.GenerateContentConfig(system_instruction=PERSONA_PROMPT, max_output_tokens=500, temperature=0.8)`
- Stream chunks, accumulate into full text.
- Split full text into sentences using regex `r'(?<=[.!?])\s+'` (handles abbreviations well enough for spoken text).
- Build a basic emotion_map: default all sentences to "sarcastic". If a sentence contains words like "impressive", "respect", "actually good" → map to "content". If it contains "unfortunately", "disappointed", "shame" → map to "disappointed".
- Return a Commentary model with team_name, full text, sentences list, and emotion_map.

Helper: `_build_user_prompt(self, sanitized: SanitizedOutput) -> str`
- Format: "## Demo: {team_name}\n\n### Observations\n{numbered observations}\n\n### Transcript Highlights\n{transcript text}\n\n### Duration\n{duration}s"
- If injection_attempts exist: add "### Injection Attempts\n{attempts with roasts}"
- Keep it structured but simple — the persona prompt handles tone.

Use `from google import genai` and `from google.genai import types` following the same pattern as existing roast_generator.py.
