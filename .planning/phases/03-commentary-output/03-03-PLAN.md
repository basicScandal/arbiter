**Create `src/commentary/qa_generator.py`** with class **QAGenerator**:

Constructor: `__init__(self, api_key: str, model: str = "gemini-2.5-flash")`
- Create genai.Client and store model name.

Method: `async def generate(self, sanitized: SanitizedOutput) -> list[QAQuestion]`
- Build a user prompt from SanitizedOutput: include observations, transcript highlights, injection attempts.
- Call `client.aio.models.generate_content()` (non-streaming is fine for 1-2 short questions) with:
  - `model=self._model`
  - `contents=user_prompt`
  - `config=types.GenerateContentConfig(system_instruction=QA_PROMPT, max_output_tokens=200, temperature=0.7)`
- Parse response: split on newlines, filter empty lines, create QAQuestion for each.
- Wrap in try/except: on error, return a single fallback question: "Could you walk us through the most interesting technical decision you made and why?" with context "fallback".

**Create `src/commentary/pipeline.py`** with class **CommentaryPipeline**:

Constructor: `__init__(self, api_key: str, voice_id: str, display_host: str = "0.0.0.0", display_port: int = 8080)`
- Create CommentaryGenerator(api_key=api_key).
- Create QAGenerator(api_key=api_key).
- Create TTSEngine(api_key=cartesia_api_key, voice_id=voice_id). Note: Cartesia API key comes from env var CARTESIA_API_KEY — read it via `os.environ.get("CARTESIA_API_KEY", "")`. If empty, log a warning but continue (TTS will degrade gracefully).
- Create DisplayServer(host=display_host, port=display_port).
- Store `_event_bus: EventBus | None = None`.
- Store `_last_sanitized: SanitizedOutput | None = None` for Q&A reference.

Method: `async def setup(self, event_bus: EventBus) -> None`
- Store event_bus reference.
- Subscribe to "observation_verified" → `self._on_observation_verified`.
- Subscribe to "qa_requested" → `self._on_qa_requested`.
- Subscribe to "tts_speaking" on audio capture mute coordination (subscribe so the audio capture component receives it — the event bus handles dispatch, no direct wiring needed here).
- Connect TTS engine: `await self._tts.connect()`. Wrap in try/except — log warning on failure, TTS degrades gracefully.
- Start display server: `await self._display.start()`.
- Log: "Commentary pipeline armed".

Method: `async def _on_observation_verified(self, event: ObservationVerified) -> None`
- Store `self._last_sanitized = event.output`.
- Generate commentary: `commentary = await self._generator.generate(event.output)`.
- Clear the display first: `await self._display.clear()`.
- Stream sentences to TTS and display in parallel. For each sentence (with index i):
  - Determine emotion from commentary.emotion_map (default "sarcastic").
  - Create a unique context_id for this commentary (once, outside loop).
  - Run TTS speak and display push concurrently:
    ```python
    await asyncio.gather(
        self._tts.speak(sentence, context_id, emotion, is_continuation=(i > 0)),
        self._display.push_commentary(sentence, event.output.team_name),
    )
    ```
  - This ensures each sentence is spoken and displayed before moving to the next (sequential sentences, parallel output channels).
- After all sentences delivered, publish CommentaryDelivered event.
- Log: "Commentary delivered for team: {team_name}".
- Wrap entire method in try/except: log any error, never crash. Commentary failure should not block the pipeline.

Method: `async def _on_qa_requested(self, event: QARequested) -> None`
- If `_last_sanitized` is None, log warning and return (no demo to question about).
- Generate questions: `questions = await self._qa_generator.generate(self._last_sanitized)`.
- For each question:
  - Speak via TTS with emotion "curious" (or fall back to default if Cartesia doesn't support it — use "neutral" as fallback).
  - Push to display as question type.
- Log: "Q&A questions delivered for team: {team_name}".
- Wrap in try/except for resilience.

Method: `async def close(self) -> None`
- Close TTS engine.
- Stop display server.
