    1. Run `uv add tenacity~=9.0` to install the dependency.

    2. Create `src/resilience/__init__.py` exporting the key names from retry.py and health.py.

    3. Create `src/resilience/retry.py` with shared tenacity retry decorator configs:
       - `GEMINI_RETRY`: retry decorator for Gemini API calls. Config:
         - `stop=stop_after_attempt(3)` for interactive paths
         - `wait=wait_exponential_jitter(initial=1, max=10, jitter=2)`
         - `retry=retry_if_exception_type((ConnectionError, TimeoutError, OSError))` -- these cover network failures. Do NOT retry on ValueError or google.genai-specific auth errors.
         - `before_sleep` callback that logs a WARNING with attempt number
       - `GEMINI_RETRY_BACKGROUND`: same as above but `stop_after_attempt(5)` for background paths (scoring, deliberation)
       - Both decorators are plain tenacity.retry instances that can be applied with `@GEMINI_RETRY` or composed dynamically.

    4. Create `src/resilience/health.py` with ServiceHealth class:
       - Constructor: `recovery_window: float = 60.0`
       - `_healthy: dict[str, bool]`, `_last_failure: dict[str, float]`, `_failure_count: dict[str, int]`
       - `mark_healthy(service)`: sets healthy True, resets failure count
       - `mark_unhealthy(service)`: sets healthy False, records time, increments failure count
       - `is_healthy(service) -> bool`: returns True if never tracked or healthy. If unhealthy, checks if `recovery_window` seconds have elapsed since last failure -- if so, returns True (allow retry). Multiply recovery_window by 2^(failure_count-1) capped at 600s for exponential backoff on repeated failures.
       - `get_status() -> dict[str, bool]`: returns current health for all tracked services
       - `failure_count(service) -> int`: returns count for a service
       - Module-level `default_health = ServiceHealth()` singleton following the EventBus pattern.
  </action>

    Run `python -c "import tenacity; print(tenacity.__version__)"` prints a 9.x version.
  </verify>
  <done>tenacity is installed. GEMINI_RETRY and GEMINI_RETRY_BACKGROUND decorators exist with exponential backoff + jitter. ServiceHealth class tracks per-component health with exponential recovery windows. Module-level default_health singleton exists.</done>

    Apply tenacity retry to all three Gemini-calling modules. The pattern is: extract the API call into a private method decorated with the retry, keeping the outer public method's existing try/except + fallback logic intact. The retry fires BEFORE the fallback -- retries attempt to succeed, and only on final failure does the existing except block catch and produce a fallback.

    1. **src/commentary/generator.py** -- CommentaryGenerator.generate():
       - Extract the streaming Gemini call (lines 61-71) into a private async method `_stream_gemini(self, user_prompt: str) -> str`
       - Decorate `_stream_gemini` with `@GEMINI_RETRY` (interactive path, 3 attempts)
       - The `generate()` method calls `_stream_gemini()` inside the existing try/except -- the retry handles transient errors, and only after 3 failures does the exception propagate to the existing fallback text logic
       - Import: `from src.resilience.retry import GEMINI_RETRY`

    2. **src/scoring/engine.py** -- ScoringEngine.score():
       - Extract the Gemini generate_content call (lines 92-100) into a private async method `_call_gemini(self, prompt: str) -> str`
       - Decorate `_call_gemini` with `@GEMINI_RETRY_BACKGROUND` (background path, 5 attempts)
       - The `score()` method calls `_call_gemini()` inside the existing try/except -- retries first, then fallback scorecard on final failure
       - Import: `from src.resilience.retry import GEMINI_RETRY_BACKGROUND`

    3. **src/memory/deliberation_engine.py** -- DeliberationEngine.deliberate():
       - Extract the Gemini generate_content call (lines 86-97) into a private async method `_call_gemini(self, prompt: str) -> DeliberationResult`
       - Decorate `_call_gemini` with `@GEMINI_RETRY_BACKGROUND` (background path, 5 attempts)
       - The `deliberate()` method calls `_call_gemini()` -- deliberation re-raises on failure (operator-triggered, not automatic), so the retry gives it 5 attempts before the raise
       - Import: `from src.resilience.retry import GEMINI_RETRY_BACKGROUND`

    Important: Do NOT change any existing fallback logic. The retry wraps the Gemini call itself. Existing try/except blocks catch what survives the retry.
