       - async def speak(self, text: str) -> None:

       - Import MacOSSayFallback from tts_fallback

       - Modify speak() method:
         - Keep existing TTSSpeaking publish at the top
         - Keep existing try block for Cartesia WebSocket speak
         - In the existing except Exception block (line 114), BEFORE logging the error, attempt fallback:
           - Log warning "Cartesia TTS failed, attempting macOS say fallback"
           - If self._fallback.available, await self._fallback.speak(sentence)
           - Else log warning "macOS say fallback not available"
           - Wrap the fallback attempt in its own try/except so it does not disrupt the finally block
         - The finally block with TTSFinished publish stays EXACTLY as-is -- this is critical for audio capture mute coordination
       - Add a _ensure_connected method that checks self._connected and if False, attempts to reconnect:
         - If self._connected, return True
         - Try await self.connect(), return True
         - On exception, log warning "TTS reconnection failed", return False
       - In speak(), call _ensure_connected() at the top. If returns False AND fallback not available, log warning and skip (but still publish TTSFinished via the finally block).
