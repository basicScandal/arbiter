```
dev = [
    "pytest>=9.0.2",
    "pytest-asyncio>=0.23",
    "pytest-timeout~=2.4",
    "pytest-xdist~=3.8",
    "vcrpy~=8.1",
    "pytest-recording~=0.13",

```

Add a `[tool.pytest.ini_options]` section with these settings:
```toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
asyncio_default_fixture_loop_scope = "function"
timeout = 30
timeout_method = "thread"
addopts = "-x --tb=short"
markers = [
    "integration: tests requiring full pipeline wiring or external dependencies",
    "slow: tests taking >5s (real API calls, cassette recording)",
]
```

Then run `uv sync --group dev` to install the new dependencies.

Key details:
- `asyncio_mode = "auto"` makes existing `@pytest.mark.asyncio` markers redundant (harmless no-ops). All async test functions are auto-detected.
- `timeout_method = "thread"` works with xdist workers and threads (unlike "signal" which only works in main thread).
- `timeout = 30` kills any hanging test after 30 seconds -- covers the unawaited-task hang scenario.
- `addopts = "-x --tb=short"` fails fast on first error with short tracebacks for readability.
  </action>

**Create `tests/helpers/__init__.py`** as an empty file (package marker).

**Create `tests/helpers/event_collector.py`** with the EventCollector class:
- Constructor takes an `EventBus` instance and calls `event_bus.subscribe_all(self._on_event)`
- `_on_event(self, event: CaptureEvent)` appends to `self.events` list and sets any matching waiter
- `wait_for(self, event_type: str, timeout: float = 5.0) -> CaptureEvent` checks if event already captured, otherwise registers an `asyncio.Event` waiter and awaits it with `asyncio.wait_for(waiter.wait(), timeout=timeout)`. Returns the last event of that type.
- `of_type(self, event_type: str) -> list[CaptureEvent]` filters captured events by type
- `count(self, event_type: str) -> int` returns count of events of a type
- Use `dict[str, asyncio.Event]` for `_waiters`. Clean up the waiter in a finally block.

**Create `tests/conftest.py`** with three fixtures:

1. `_reset_singletons` (autouse=True, function scope):
   - Import `src.capture.event_bus` as `event_bus_module` and `src.resilience.health` as `health_module`
   - Before yield: replace `event_bus_module.default_bus` with fresh `EventBus()`, replace `health_module.default_health` with fresh `ServiceHealth()`, set `GeminiRateLimiter._instance = None`
   - After yield: same three resets for post-test cleanup
   - This replaces the local autouse fixture in test_rate_limiter.py (which can remain -- it's harmless, just redundant)

2. `event_bus` fixture (function scope):
   - Returns a fresh `EventBus()` instance for tests that need an isolated bus

3. `event_collector` fixture (function scope):
   - Takes `event_bus` fixture as parameter
   - Returns `EventCollector(event_bus)` bound to that bus

4. `vcr_config` fixture (module scope):
   - Returns dict with `filter_headers` (authorization, x-api-key, x-goog-api-key, anthropic-api-key), `record_mode` "none", `cassette_library_dir` "tests/cassettes", `decode_compressed_response` True

Critical implementation details:
- The singleton reset MUST replace the module attribute, not just clear subscribers. Code that does `from src.capture.event_bus import default_bus` at import time gets a reference, but code that does it at call time (like DemoMachine.__init__) reads the module attribute fresh. Replacing the module attribute handles both patterns correctly because test_qa.py imports default_health locally inside test methods.
- EventCollector.wait_for() is safe to call after bus.publish() because asyncio.create_task() schedules but doesn't execute until current coroutine yields. So the sequence publish() -> wait_for() -> yield (on waiter.wait()) is race-free.
