  <action>
    **src/capture/gemini_session.py**: Gemini Live API session management:

    - `GeminiSession` class:
      - `__init__(self, config: CaptureConfig, event_bus: EventBus, in_queue: asyncio.Queue)`: Store config, event bus, input queue. Initialize:
        - `_client = genai.Client(api_key=config.gemini_api_key, http_options={"api_version": "v1beta"})`
        - `_session: AsyncSession | None = None`
        - `_resumption_handle: str | None = None`
        - `_stop_event = asyncio.Event()`
        - `_observations: list[str] = []` — accumulates Gemini text observations for the current demo

      - `_build_config(self) -> types.LiveConnectConfig`: Build config with:
        - `response_modalities=["TEXT"]`
        - `system_instruction` — "You are observing a live hackathon demo. Describe what you see on screen (slides, code, demos, terminal output) and what the presenter is saying. Provide structured, factual observations. Do not speculate or editorialize."
        - `input_audio_transcription=types.AudioTranscriptionConfig()`
        - `context_window_compression=types.ContextWindowCompressionConfig(trigger_tokens=config.compression_trigger_tokens, sliding_window=types.SlidingWindow(target_tokens=config.compression_target_tokens))`
        - `session_resumption=types.SessionResumptionConfig(handle=self._resumption_handle)`

      - `async _send_loop(self) -> None`: Consume from in_queue, send to Gemini:
        - Loop while not stopped
        - `msg = await asyncio.wait_for(self.in_queue.get(), timeout=1.0)` — use timeout to check stop event periodically
        - If msg.mime_type starts with "audio/": `await self._session.send_realtime_input(audio=types.Blob(data=msg.data, mime_type=msg.mime_type))`
        - If msg.mime_type starts with "image/": `await self._session.send_realtime_input(media={"mime_type": msg.mime_type, "data": base64.b64encode(msg.data).decode()})` (video frames need base64 encoding)
        - Catch `asyncio.TimeoutError` and continue (just checking stop event)

      - `async _receive_loop(self) -> None`: Receive from Gemini session:
        - Loop while not stopped
        - `turn = self._session.receive()`
        - `async for response in turn`:
          - If `response.session_resumption_update` and `.resumable` and `.new_handle`: store `self._resumption_handle = update.new_handle`
          - If `response.text`: append to `_observations`, log the observation
          - If `response.server_content` and `.input_transcription`: create `TranscriptSegment(text=..., timestamp=time.time())`, publish `TranscriptReceived(segment=...)` to event bus
          - If `response.server_content` and `.turn_complete`: log turn boundary
        - Handle `google.genai.errors.ServerError` or connection errors — log and attempt reconnect

      - `async run(self) -> None`: Main entry point:
        - Connect: `async with self._client.aio.live.connect(model=config.gemini_model, config=self._build_config()) as session:`
        - Store `self._session = session`
        - Run `_send_loop()` and `_receive_loop()` concurrently with `asyncio.TaskGroup`
        - On connection error or GoAway: reconnect with stored resumption handle

      - `async stop(self) -> None`: Set stop event, return accumulated observations.

      - `get_observations(self) -> list[str]`: Return copy of `_observations`.

      - `clear_observations(self) -> None`: Clear `_observations` list (call between demos).

    Use proper logging throughout. Handle connection errors gracefully -- log and attempt reconnect. The session should be resilient to transient failures.
