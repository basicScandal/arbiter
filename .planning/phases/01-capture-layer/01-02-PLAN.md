    1. **src/capture/key_frames.py**: Key frame detection via histogram comparison:
       - `KeyFrameDetector` class:
         - `__init__(self, threshold: float = 0.4)`: Store threshold, initialize `_prev_hist` as None.
         - `check(self, frame: numpy.ndarray) -> bool`: Compare frame's grayscale histogram against previous frame. Use `cv2.calcHist`, `cv2.normalize`, `cv2.compareHist` with `cv2.HISTCMP_CORREL`. If correlation < threshold, it's a key frame. First frame is always a key frame. Store current histogram as `_prev_hist` for next comparison. Return True if key frame.
         - `reset(self) -> None`: Clear `_prev_hist` (call between demos).

    2. **src/capture/camera.py**: Async camera frame capture:
       - `CameraCapture` class:
         - `__init__(self, config: CaptureConfig, event_bus: EventBus, out_queue: asyncio.Queue)`: Store config, event bus, output queue. Create `KeyFrameDetector(threshold=config.key_frame_threshold)`. Create `asyncio.Event` for stop signal.
         - `_capture_and_encode(self, cap: cv2.VideoCapture) -> tuple[FrameData, numpy.ndarray] | None`: Synchronous method (runs in thread). Calls `cap.read()`. If frame captured: convert BGR->RGB with `cv2.cvtColor`, create PIL.Image, thumbnail to `[config.frame_max_dimension, config.frame_max_dimension]`, encode as JPEG to BytesIO, base64-encode. Return `(FrameData(...), raw_numpy_frame)` tuple. Return None if read fails.
         - `async run(self) -> None`: Main capture loop:
           - Open camera: `cap = await asyncio.to_thread(cv2.VideoCapture, config.camera_device_index)`
           - Verify `cap.isOpened()`, raise RuntimeError if not
           - Loop while not stopped:
             - `result = await asyncio.to_thread(self._capture_and_encode, cap)`
             - If result is None, break
             - Check key frame: `is_kf = self.key_frame_detector.check(raw_frame)`, set `frame_data.is_key_frame = is_kf`
             - Publish `FrameCaptured(frame=frame_data)` to event bus
             - If is_kf, also publish `KeyFrameDetected(frame=frame_data)` to event bus
             - Put the media chunk (JPEG data with mime_type "image/jpeg") into out_queue using `put_nowait`, catch `QueueFull` and silently drop (log warning)
             - `await asyncio.sleep(1.0 / config.frame_rate)` — ~1 FPS default
           - Finally: `cap.release()`
         - `async stop(self) -> None`: Set stop event. Reset key frame detector.

    Use proper logging (import logging, get logger for module). All blocking OpenCV calls MUST go through `asyncio.to_thread()`.

      - `mute(self) -> None`: Set `_muted = True`. Log "Audio muted".

      - `is_muted(self) -> bool`: Return `_muted`.
      - `async run(self) -> None`: Main audio capture loop:
        - Initialize PyAudio: `pya = pyaudio.PyAudio()`
        - Open stream: `stream = await asyncio.to_thread(pya.open, format=pyaudio.paInt16, channels=config.audio_channels, rate=config.audio_sample_rate, input=True, input_device_index=config.audio_device_index, frames_per_buffer=config.audio_chunk_size)`
        - Note: `input_device_index=None` means PyAudio uses system default
        - Loop while not stopped:
          - `data = await asyncio.to_thread(stream.read, config.audio_chunk_size, exception_on_overflow=False)`
          - If `_muted`, skip processing (continue loop, discarding audio data)
          - Create media chunk: `MediaChunk(mime_type="audio/pcm", data=data)`
          - Put into out_queue using `put_nowait`, catch `QueueFull` and silently drop
          - No sleep needed -- `stream.read()` blocks naturally for the chunk duration (~32ms at 512 samples / 16kHz)
        - Finally: `stream.close()`, `pya.terminate()`
      - `async stop(self) -> None`: Set stop event.

    Use proper logging. All PyAudio blocking calls (`pya.open`, `stream.read`) MUST go through `asyncio.to_thread()`. Handle `exception_on_overflow=False` in `stream.read()` to prevent crashes on buffer overrun.
