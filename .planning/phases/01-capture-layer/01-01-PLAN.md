  <action>
    1. **src/capture/event_bus.py**: Build a lightweight async event bus (do NOT use asyncio-signal-bus -- hand-roll with asyncio.Queue for simplicity):
       - `EventBus` class with:
         - `_subscribers: dict[str, list[Callable]]` — maps event_type to list of async callback functions
         - `subscribe(event_type: str, callback: Callable) -> None` — register a callback for an event type
         - `unsubscribe(event_type: str, callback: Callable) -> None` — remove a callback
         - `publish(event: CaptureEvent) -> None` — call all subscribers for event.event_type. Use `asyncio.create_task()` for each callback so publishing is non-blocking. Log errors from callbacks but don't propagate (bus must be resilient).
         - `subscribe_all(callback: Callable) -> None` — register callback for ALL event types (useful for logging)
       - Import and use CaptureEvent from models.py.
       - Add module-level `default_bus = EventBus()` singleton for convenience.

    2. **src/capture/demo_machine.py**: Build the demo lifecycle state machine using python-statemachine:
       - `DemoMachine(StateMachine)` with states:
         - `idle` (initial=True) — no demo running
         - `capturing` — demo in progress, capture active
         - `stopped` — demo ended, data available
       - Transitions:
         - `start_demo`: idle -> capturing
         - `stop_demo`: capturing -> stopped
         - `reset`: stopped -> idle
       - Store `current_session: DemoSession | None` as instance attribute.
       - `on_enter_capturing(team_name: str)`: Create new DemoSession with team_name and started_at=time.time(). Publish DemoStarted event to event bus.
       - `on_enter_stopped()`: Set current_session.stopped_at. Publish DemoStopped event with team_name and duration.
       - `on_enter_idle()`: Clear current_session to None.
       - Constructor takes `event_bus: EventBus` parameter (default to `default_bus`).
       - Note: python-statemachine 2.x uses `on_enter_state` pattern. Check that async entry/exit actions work correctly. If python-statemachine does not support async on_enter callbacks natively, use synchronous callbacks that schedule coroutines via `asyncio.get_event_loop().create_task()` or wrap with `asyncio.run_coroutine_threadsafe()`. Alternatively, make the event bus publish synchronous (schedule async callbacks internally).

    Test the integration by writing a small script at the bottom of demo_machine.py guarded by `if __name__ == "__main__"` that:
    - Creates an EventBus and DemoMachine
    - Subscribes a print callback to "demo_started" and "demo_stopped"
    - Runs start_demo("TestTeam") -> stop_demo() -> reset()
    - Verifies transitions complete without error
