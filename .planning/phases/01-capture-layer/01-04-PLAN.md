<objective>
Wire all capture components into a single pipeline orchestrator and create the application entry point. This is the integration plan -- it connects camera, audio, Gemini session, state machine, and operator CLI into a working system.

Purpose: Individual components work in isolation but need orchestration to function as a system. The pipeline manages the lifecycle: when operator starts a demo, camera/audio tasks spin up and feed the Gemini session. When operator stops, tasks clean up and data is preserved. This is what makes the capture layer actually usable.

Output: pipeline.py (orchestrator) and main.py (entry point). A working end-to-end capture system that an operator can control.
</objective>

  <action>
    1. **src/capture/pipeline.py**: Pipeline orchestrator that wires components together:
       - `CapturePipeline` class:
         - `__init__(self, config: CaptureConfig)`: Create shared components:
           - `self.event_bus = EventBus()`
           - `self.media_queue = asyncio.Queue(maxsize=config.max_queue_size)` — shared between producers and Gemini consumer
           - `self.demo_machine = DemoMachine(event_bus=self.event_bus)`
           - `self.camera = CameraCapture(config=config, event_bus=self.event_bus, out_queue=self.media_queue)`
           - `self.audio = AudioCapture(config=config, event_bus=self.event_bus, out_queue=self.media_queue)`
           - `self.gemini = GeminiSession(config=config, event_bus=self.event_bus, in_queue=self.media_queue)`
           - `self.cli = OperatorCLI(demo_machine=self.demo_machine)`
           - `self._capture_tasks: list[asyncio.Task] = []`

         - `async _on_demo_started(self, event: DemoStarted) -> None`:
           - Log "Demo started for team: {event.team_name}"
           - Start capture tasks: create asyncio tasks for `self.camera.run()`, `self.audio.run()`, `self.gemini.run()`
           - Store task references in `self._capture_tasks`

         - `async _on_demo_stopped(self, event: DemoStopped) -> None`:
           - Log "Demo stopped for team: {event.team_name}, duration: {event.duration:.1f}s"
           - Stop all capture components: `await self.camera.stop()`, `await self.audio.stop()`, `await self.gemini.stop()`
           - Cancel and await all capture tasks (handle CancelledError)
           - Clear `self._capture_tasks`
           - Store Gemini observations in demo session (access via demo_machine.current_session)
           - Print summary: number of key frames captured, number of transcript segments, duration

         - `async run(self) -> None`:
           - Subscribe to events: `self.event_bus.subscribe("demo_started", self._on_demo_started)`, same for "demo_stopped"
           - Subscribe a logging callback to all events: `self.event_bus.subscribe_all(self._log_event)`
           - Print startup banner: "Arbiter Capture Layer v0.1", "Type 'help' for commands"
           - Run the operator CLI: `await self.cli.run()` — this blocks until operator quits
           - On exit: clean up any running capture tasks

         - `async _log_event(self, event: CaptureEvent) -> None`: Log event type and timestamp at DEBUG level.

    2. **src/main.py**: Entry point:
       - `def main()`:
         - Load dotenv: `from dotenv import load_dotenv; load_dotenv()`
         - Load config: `config = load_config()` (this will raise if GEMINI_API_KEY missing)
         - Create pipeline: `pipeline = CapturePipeline(config)`
         - Run: `asyncio.run(pipeline.run())`
       - Guard with `if __name__ == "__main__": main()`

    The pipeline is the glue. It does NOT contain business logic -- it connects components and manages their lifecycle. Keep it thin.
